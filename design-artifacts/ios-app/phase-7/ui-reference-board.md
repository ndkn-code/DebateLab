# Phase 7 UI Reference Board

## Analysis Progress

Image: `ui-reference/analysis-progress-reference.png`

Prompt:

> Create a polished raster reference board for the Thinkfy mobile analysis progress flow after transcription. Show feedback request confirmation with Credits cost, queued/analyzing progress, and recoverable analysis failed/timeout retry state. Use Thinkfy calm blue, native iOS ergonomics, Duolingo-style momentum, and Brilliant-style clarity.

Takeaways:

- Show Credits cost before paid analysis starts.
- Keep queue/progress states calm and explicit.
- Failure states should preserve trust: transcript saved, retry available.

## Feedback Result

Image: `ui-reference/feedback-result-reference.png`

Prompt:

> Create a polished raster reference board for the Thinkfy mobile feedback result screen. Show score/band hero, category cards, strengths/improvements, and transcript annotations. Use Thinkfy calm blue, compact iOS UI, Duolingo-style motivation, and Brilliant-style learning feedback.

Takeaways:

- Score and band should be visible first, with actionable summary copy.
- Category cards must stay compact enough for mobile scanning.
- Transcript annotations should feel coachable, not punitive.

## History Detail

Image: `ui-reference/history-detail-reference.png`

Prompt:

> Create a polished raster reference board for Thinkfy mobile practice history and history detail. Show empty history, populated history list, and history detail with score, transcript, verdict/clash-map equivalent, and actions for coach/practice again.

Takeaways:

- History needs both empty and populated states.
- Detail screens should reuse feedback hierarchy without becoming a full web clone.
- Coach/practice-again actions belong near the result context.
