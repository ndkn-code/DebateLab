## Prompt

Create one polished UI reference board showing three iPhone screens for Thinkfy mobile authentication: welcome/sign-in, provider loading/session restore, and signed-in diagnostic status. Style should combine Duolingo-like friendly motivation and progress energy with Brilliant-like clarity, calm learning focus, and crisp educational product polish.

Visual system: Thinkfy calm blue palette, white surfaces, soft navy text, bright but restrained green progress accents, small celebratory learning details, 8px radius cards/buttons, no purple gradient theme, no beige/brown theme.

Screens: Screen 1 has Thinkfy brand, student speaking practice value, native Google button and native Apple sign-in button. Screen 2 shows secure session restore, subtle progress indicator, tiny streak/XP hints. Screen 3 shows signed-in diagnostic: user identity, auth provider, API smoke status, token refresh status, and sign out action.

## Phase 2 Takeaways

- Keep the auth entry screen focused on two native provider actions and a short learning-oriented value statement.
- Use Thinkfy calm blue as the dominant brand signal, with green only as a status/progress accent.
- Make the signed-in view explicitly diagnostic: user identity, provider, token/API status, and sign-out should be visible without needing the Phase 3 navigation shell.
- Avoid marketing-page composition; this is a mobile app surface and should feel immediately usable.
