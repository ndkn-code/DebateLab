# Phase 6 UI Reference Board

## Upload And Transcription Progress

Image: `ui-reference/upload-transcription-progress-reference.png`

Prompt:

> Create a polished raster reference board for the Thinkfy iOS app upload and transcription progress flow after a student finishes a speaking practice recording. Use Thinkfy calm blue, crisp white surfaces, native iOS ergonomics, Duolingo-style momentum, and Brilliant-style learning clarity. Show three iPhone screens: automatic audio upload, Deepgram transcription in progress, and transcript ready with confidence/word-count badges.

Takeaways:

- Keep the student oriented with a simple status timeline: upload, transcript, ready.
- Use positive momentum copy, but keep privacy/status information explicit.
- Preserve compact cards and safe-area spacing so the native tab bar never competes with the workflow.

## Transcript Review And Recovery

Image: `ui-reference/transcript-review-reference.png`

Prompt:

> Create a polished raster reference board for Thinkfy mobile transcription review and error recovery after a speaking practice audio upload. Show transcript review, no-speech warning, upload failed retry, and transcription failed retry. Use Thinkfy calm blue, compact native iOS UI, Duolingo-style encouraging microcopy, and Brilliant-style clear learning feedback.

Takeaways:

- A failed upload/transcription should reassure the student that the local recording is preserved.
- Transcript readiness is not the same as feedback readiness; Phase 7 owns scoring and analysis.
- Error states should offer retry without forcing re-recording unless the transcript is genuinely unusable.
